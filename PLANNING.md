## 📚 Technology Stack

### Backend Stack
- **Runtime**: Bun 1.0+ (TypeScript-native, fast startup, built-in SQLite)
- **Web Framework**: Bun native HTTP server + Hono.js for routing
- **Database**: SQLite with Better-SQLite3 driver
- **Real-time**: Native WebSocket support in Bun
- **Validation**: Zod for runtime type validation
- **Testing**: Bun test runner

**Rationale**: Bun provides excellent performance, native TypeScript support, and simplified tooling. SQLite ensures zero-config local storage with excellent performance for single-user applications.

### Frontend Stack
- **Framework**: React 18 with TypeScript
- **Build Tool**: Vite (Bun compatible)
- **Styling**: Tailwind CSS (v0 optimized)
- **State Management**: Zustand (lightweight, TypeScript-first)
- **Data Fetching**: TanStack Query (React Query)
- **Real-time**: Native WebSocket API
- **UI Components**: Shadcn/ui (v0 preferred, excellent TypeScript support)
- **Icons**: Lucide React

**Rationale**: React + TypeScript provides excellent v0 compatibility. Zustand offers simpler state management than Redux. TanStack Query handles server state elegantly with caching and synchronization.

### Development Tools
- **Package Manager**: Bun (faster than npm/yarn)
- **Type Checking**: TypeScript strict mode
- **Linting**: ESLint with TypeScript rules
- **Formatting**: Prettier
- **Git Hooks**: Husky + lint-staged

## 🌐 API Architecture

### RESTful API Structure
```typescript
// Base URL: http://localhost:3000/api

// Life Areas
GET    /api/life-areas                    # List all life areas
POST   /api/life-areas                    # Create life area
GET    /api/life-areas/:id               # Get specific life area
PUT    /api/life-areas/:id               # Update life area
DELETE /api/life-areas/:id               # Archive life area

// Goals
GET    /api/goals                        # List all goals
GET    /api/goals/by-life-area/:id       # Goals for specific life area
POST   /api/goals                        # Create goal
GET    /api/goals/:id                    # Get specific goal
PUT    /api/goals/:id                    # Update goal
DELETE /api/goals/:id                    # Archive goal

// Projects
GET    /api/projects                     # List all projects
GET    /api/projects/by-goal/:id         # Projects for specific goal
POST   /api/projects                     # Create project
GET    /api/projects/:id                 # Get specific project
PUT    /api/projects/:id                 # Update project
DELETE /api/projects/:id                 # Archive project

// Tasks
GET    /api/tasks                        # List all tasks
GET    /api/tasks/by-project/:id         # Tasks for specific project
POST   /api/tasks                        # Create task
GET    /api/tasks/:id                    # Get specific task
PUT    /api/tasks/:id                    # Update task
DELETE /api/tasks/:id                    # Archive task

// Hierarchy Operations
GET    /api/hierarchy/:type/:id/children # Get direct children
GET    /api/hierarchy/:type/:id/tree     # Get full subtree
POST   /api/hierarchy/reorder            # Reorder items
GET    /api/hierarchy/search?q=:query    # Cross-hierarchy search

// Data Management
GET    /api/export?format=:format        # Export data (json|csv)
POST   /api/import                       # Import data
POST   /api/backup                       # Create backup
GET    /api/backup/list                  # List backups
POST   /api/backup/restore/:id           # Restore backup

// WebSocket
WS     /ws                               # Real-time updates
```

## ⚡ Real-time Synchronization

### WebSocket Event Types
```typescript
type WebSocketMessage = 
  | { type: 'item_created'; payload: HierarchyItem & { entityType: string } }
  | { type: 'item_updated'; payload: HierarchyItem & { entityType: string } }
  | { type: 'item_deleted'; payload: { id: string; entityType: string } }
  | { type: 'status_changed'; payload: { id: string; entityType: string; status: string } }
  | { type: 'reorder'; payload: { parentId: string; entityType: string; newOrder: string[] } }
  | { type: 'connection_status'; payload: { connected: boolean; clientCount: number } };
```

### Implementation Strategy
- **Auto-reconnection**: Exponential backoff with max delay
- **Event Broadcasting**: Server notifies all connected clients
- **Optimistic Updates**: UI updates immediately, reverts on error
- **Conflict Resolution**: Last-write-wins with timestamp comparison
- **Heartbeat**: Ping/pong to maintain connection health
